FAQ

Just some quick FAQ and Troubleshooting info for the Advanced BBCode Box Mod.

Troubleshooting

My Installation Failed, what do I do now?

The first thing to check is to make sure all files were uploaded to the proper directories. If you manually edited files you need to verify all edits were done correctly. There is a copy of the original un-edited files in the extras directory, and it may help to use a file comparison tool such as WinMerge to compare RN v2.50 Pre-edited files with extras/RN v2.50 Original files to see all the edits side by side.

The ABBC editor is not functioning properly, what now?

Review step one of installation, making sure the core mod files were properly uploaded, as well as the Forum Language and Posting/Private Message edits/uploads.

Why do certain bbcode's disappear upon posting or preview?

It's likely you either didn't upload a modified bbcode.tpl to your theme (or the default forum templates within modules/Forums/templates/*) or made mistakes during subsequent edits to the file.

My bbcode's are not processed and have weird codes added to them

If upon posting a given bbcode you end up with something similar to below, where the code is not processed and has codes added to it; the issue is likely within function bbencode_second_pass of bbcode.php. If you manually edited that file confirm all edits were done correctly. Note that some cases may occur by user error, by providing values within the bbcode that are invalid. This occurs because some matches with function bbencode_first_pass are overly aggressive and some more specific matches take place during bbencode_second_pass. 

[somecode:234910043458849301]my content[/somecode:234910043458849301]

I've tried everything and it still won't work, should I give up?

Maybe...lol. If installing this mod is beyond your skill set, I would consider installing for you for a nominal fee; provided you are running RavenNuke™ v2.5 and your theme isn't ancient and /or full of validation errors. Send me a PM on Raven's site if you need paid assistance.

FAQ

Will you add support for video site XYZ for me?

NO! This was exhausting! There are instructions in Customization for adding/altering custom buttons, but if doing so is beyond your skill-set, you will have to wait for a future release. While the phpbb3 version of this mod supports a TON of different video sites, don't expect that type of functionality from this mod anytime soon. Although I would like to add support for a handful of the most popular sites in a future version, as well as a configuration panel to enable/disable particular sites. No promises, but it's on my internal wish list.

Is the code produced by the mod 100% XHTML valid?

No. But it is valid for the most part. The proprietary attribute allowfullscreen is required for audio/video iframe embeds, unless you want an ugly border in some browsers. The [stream] and [flash] bbcode's were untouched from the previous release of ABBC Box so I can't speak to how valid they are. It is also possible for users to generate validation warnings by improper use of bbcode. Many of the bbcode tags use <span> while some others by nature must use <div>. Since <div> is not valid within <span> users can generate validation errors but they should not be serious enough to effect the layout of your site. While warnings may be generated for <div> (which is usually serious) it is actually the <span> tag which has errors. Short of a complex regex solution to find improperly nested bbcode, I'm not sure too much can be done about it easily. It's a limitation of the mod and a side-effect of giving users more control over formatting their posts.

HTML5 Ready?

Mostly, but there are still some deprecated table codes present within the mod that would require cleanup. Although lacking a true RN html5 test environment at this time, I can't specify with any assurance how much work would be involved. The code highlighter integration does wrap code with class names recommended in HTML5: "language-html", "language-php", etc..

How does it work on Mobile Devices?

I have no idea, but something tells me it would be difficult to use on a mobile device simply because the buttons are so small. I had mobile devices in mind when designing the editor and will likely try a mobile version later down the road, with big buttons and only the most important features... But that will be intended for a mobile theme which isn't finished yet... hopefully my next project!
