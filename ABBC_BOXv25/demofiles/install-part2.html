Choosing Editor Themes

Default Editor Themes

The Advanced BBCode Box mod comes with four editor flavors, black, light blue, crimson, and sand. If you wish to use the default light blue editor you will not need to upload any additional files to your theme; and you can skip to the next step below. If you wish to use the black, crimson, or sand themed editors for your theme, locate the appropriate directory within ABBC_BOXv25/extras/Editor_Themes/ and upload to themes/YOUR_THEME/style/.

So for instance, to use the crimson themed editor, upload the contents of:
ABBC_BOXv25/extras/Editor_Themes/crimson to themes/YOUR_THEME/style/

Note that there are copies of both bbcode_box.css and bbcode_addons.css within the example editor themes
ABBC_BOXv25/extras/Editor_Themes/ANY_THEME, but only bbcode_box.css is required for the editor.

If you do not wish to override user styles do not upload bbcode_addons.css, as these are just recommended "companions" to the editor theme and may not be appropriate for your theme, background color, etc. Just be aware the color swatches for the editor are declared within bbcode_addons.css for clarity, since they must match the classes used to wrap the text; which are also defined in that file.

Custom Editor Themes

Custom editor themes can be created with a little time, patience, and CSS know-how. Difficulty will depend on how many changes are desired. The Advanced BBCode Box mod is powered by two CSS files, which by default are loaded from mods/bbcode_box/bbcode_box.css and mods/bbcode_box/bbcode_addons.css, producing the default light blue editor intended for use on light backgrounds. bbcode_box.css is only loaded for users and styles the editor, and bbcode_addons.css loads for all visitors to style some of the code produced by the mod. To override these files simply upload a copy of bbcode_box.css and/or bbcode_addons.css to themes/YOUR_THEME/style/ and customize as needed. Some more tips on customization will be provided a little later on in the document...

function mozWrap(txtarea, open, close)

Posting Preview Edits

This edit is only required in some themes, making sure that {MESSAGE} is NOT wrapped in an inline element such as <span></span>. This issue is present in the default RN subSilver template, but in none of the other core RN themes.


  • yoursite.com
  • themes
  • ANY_THEME
  • forums
  • posting_preview.tpl
Find select
<span class="postbody">{MESSAGE}</span>
Replace With select
<div class="postbody">{MESSAGE}</div>

Posting Topic Review Edits

Some versions of Internet Explorer do not handle nested iframes well, which is the primary method of audio and video embeds... So a simple edit to remove the iframe and replace with a fixed height div.


  • yoursite.com
  • themes
  • ANY_THEME
  • forums
  • posting_topic_review.tpl
Find select
<iframe width="100%" height="300" src="{U_REVIEW_TOPIC}">
Replace With select
<div id="ftopic-review">
Find select
</iframe>
Replace With select
</div>

View Topic Body Edits

Only Required for some themes

This edit is only required for some themes, just need to make sure the main body of the post is NOT wrapped in a span, as several block level elements will ultimately be nested there. While your viewtopic_body.tpl may differ in appearance, just check for {postrow.MESSAGE} and {postrow.SIGNATURE} and make sure they are nested in a block level element, such as a <div>


  • yoursite.com
  • themes
  • ANY_THEME
  • forums
  • viewtopic_body.tpl
Find select
<span class="postbody">{postrow.MESSAGE}{postrow.SIGNATURE}</span>
Replace With select
<div class="postbody">{postrow.MESSAGE}{postrow.SIGNATURE}</div>
